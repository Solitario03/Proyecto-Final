Análisis de precios y disponibilidad de libros académicos

Proyecto de integración y análisis de datos

Curso: Lenguaje de Programación II

Modalidad: Trabajo colaborativo con GitHub

Objetivo: recopilar, integrar y analizar información de libros académicos desde múltiples fuentes para apoyar la toma de decisiones.

Problema

  • Los libros académicos se encuentran en múltiples plataformas.
  • La información está dispersa y no es uniforme.
  • Comparar precios y disponibilidad manualmente es ineficiente.
Se requiere una solución automatizada que consolide la información.

Trabajo colaborativo

  • Integrante 1: Web Scraping de plataformas web (HTML).
  • Integrante 2: Consumo de APIs académicas (OpenLibrary, OpenAlex, Crossref).
  • Integrante 3: Integración, limpieza, análisis y visualización de datos.
El proyecto se desarrolló siguiendo un flujo de trabajo colaborativo, donde cada etapa depende de la anterior.

Fuentes de datos utilizadas

  • OpenLibrary
  • OpenAlex
  • Crossref
El uso de múltiples fuentes permitió enriquecer y validar la información.

Integración y limpieza de datos

  • Normalización de nombres de columnas.
  • Unificación de estructuras.
  • Eliminación de datos inconsistentes.
  • Identificación de la fuente de cada registro.
Este proceso fue realizado en Python utilizando la librería Pandas.

Conclusiones

  • La automatización reduce el esfuerzo manual.
  • Las APIs y el scraping se complementan.
  • La integración mejora la calidad del análisis.
  • El trabajo colaborativo fue clave para el éxito del proyecto.

Cierre

Este proyecto demuestra cómo el análisis de datos puede apoyar la toma de decisiones en el ámbito académico.
